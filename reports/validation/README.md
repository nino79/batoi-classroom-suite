# Validation Reports

This directory holds Beta validation reports generated by `scripts/validate-beta.sh`.

## Contents

| File | Description |
|------|-------------|
| `report.md` | Summary report with command results and environment overview |
| `environment.json` | Host environment metadata (kernel, distribution, hypervisor, tools) |
| `inventory.json` | `bcs inventory --output json` |
| `inventory.yaml` | `bcs inventory --output yaml` |
| `doctor.txt` | `bcs doctor` output |
| `timings.json` | Per-command exit codes and execution durations |
| `*_stdout.txt` | Raw stdout for each `bcs` command |
| `*_stderr.txt` | Raw stderr for each `bcs` command |

## Workflow

```bash
# From the repository root:
./scripts/validate-beta.sh

# Artifacts are written to reports/validation/.
# Archive them with:
./scripts/collect-artifacts.sh
```

## CI Integration

In a CI pipeline, run `validate-beta.sh` after the `bcs` package is installed.
The `report.md` and `timings.json` files can be published as build artifacts.

See `docs/BETA_VALIDATION_AUTOMATION.md` for design and extension details.
