# Processes

This directory documents how BCS actually operates day to day, as opposed to `CONTRIBUTING.md`, which is the short, contributor-facing entry point. If `CONTRIBUTING.md` says "how to get started," these documents say "here is exactly how it works, end to end."

## Contents

| Document | Covers |
|---|---|
| [development-workflow.md](development-workflow.md) | Branching model, review requirements, CI expectations, from issue to merge. |
| [release-process.md](release-process.md) | Versioning policy, release checklist, and what triggers a release. |

## Relationship to CONTRIBUTING.md

`CONTRIBUTING.md` is intentionally short and links here for depth — keep it that way. If a detail is added to these process documents, check whether `CONTRIBUTING.md` needs a one-line pointer update, not a restatement.
